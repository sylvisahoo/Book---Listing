# Project Boundaries

1. Do not leave `print`, `console.log`, or debug statements in production code.
2. Follow clean code principles and maintain readable code structure.
3. Use meaningful and consistent naming conventions for files, variables, functions, and classes.
4. Write reusable and modular code; avoid duplication (DRY principle).
5. Handle all errors and exceptions properly.
6. Do not hardcode sensitive information such as API keys, passwords, or secrets.
7. Ensure all new features are tested before merging code.
8. Follow the project's coding standards and folder structure.
9. Do not commit any code without my permission.
